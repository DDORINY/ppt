발표 흐름에 맞춘 전체 목차

실제 발표 자료 순서를 기준으로 웹 포트폴리오를 재구성했습니다. 각 챕터는 스크롤 기반으로 이어지며, 프로젝트의 배경부터 기술 구현, 확장 비전까지 단계적으로 보여줍니다.

Project Overview

도로 위 위험을 더 빠르게 발견하고 대응하기 위한 프로젝트

도로 낙하물은 교통 흐름을 방해하고 대형 사고로 이어질 수 있는 위험 요소입니다. 기존에는 시민 신고와 CCTV 수동 모니터링에 크게 의존했기 때문에 즉각적 대응이 어렵고, 그 사이 2차 사고 가능성이 높아지는 문제가 있었습니다.

Project Goals

탐지에서 끝나지 않는 운영형 AI 시스템 구축

이번 프로젝트의 목표는 단순히 객체를 탐지하는 모델을 만드는 것이 아니라, 탐지 결과가 저장되고 관리자 대응으로 이어지며 실제 서비스처럼 동작하는 시스템을 만드는 것이었습니다.

Needs Assessment

프로젝트가 필요한 이유를 수요 관점에서 검토

단순한 아이디어 제안이 아니라, 실제 사고 위험과 공공 활용 가능성을 고려해 시스템 필요성을 정리했습니다.

4-Week Schedule

기획부터 발표까지 4주 일정

짧은 기간 안에 기획, 설계, 기능 구현, AI 모델 개발, 발표 준비까지 순차적으로 진행했습니다.

System Structure

사용자, AI, 관리자 기능을 연결하는 전체 구조

시스템은 단순 파이프라인이 아니라 사용자 입력, AI 분석, 데이터 저장, 관리자 대응, 비교분석까지 이어지는 구조로 설계했습니다.

Team Roles

역할 분담을 통해 기능과 개발 흐름을 병행

프론트엔드, 백엔드, AI 개발, 관리자 기능을 병렬로 진행하기 위해 역할을 분담하고 통합했습니다.

Differentiators

단순 탐지를 넘는 통합형 AI 시스템

저희 프로젝트는 객체 탐지 그 자체보다, 탐지된 정보를 어떻게 저장하고 전달하며 비교하고 확장하는가에 더 초점을 맞췄습니다.

Backend & DB Architecture

서비스 운영을 고려한 웹 시스템 구조 설계

Flask 기반 백엔드, MySQL 데이터베이스, 관리자 기능, AI 분석 이력을 중심으로 웹 서비스 구조를 설계했습니다.

Smart Assist

AI 기반 스마트 보조 시스템

업로드된 이미지나 영상은 단순 저장에 그치지 않고, 위치 정보와 AI 분석 결과를 바탕으로 신고 문서 작성과 관리 효율을 높이는 방향으로 확장됩니다.

AI Model Development

비교분석으로 선택 근거를 만드는 AI 개발 과정

여러 모델을 동일한 데이터 기준으로 비교하면서, 어떤 모델이 더 적합한지 판단할 수 있는 구조를 만들었습니다.

프로젝트 시연
Interactive Chart Area

기능별 모델 적용 전략

옵티마이저 비교를 통해 기준 모델을 선정한 뒤, 실제 서비스에서는 기능 목적에 따라 서로 다른 모델을 적용했습니다.

01

YOLOv8 (SGD)

기본 탐지 / 운영 기준 모델

  • 옵티마이저 비교 결과 가장 빠른 수렴 속도와 높은 mAP50-95를 보여 기준 모델로 선정
  • 신고 등록 및 기본 분석 흐름에서 가장 실용적인 모델
  • 실시간 서비스 운영에 적합한 속도와 안정성을 확보
02

RT-DETR

비교분석 / 대안 모델 검토

  • YOLO 계열과 다른 구조의 탐지 모델로 비교분석 신뢰도 강화
  • 동일 데이터에서 다른 탐지 특성을 확인할 수 있어 관리자 비교 기능에 적합
  • 단일 모델 판단이 아닌 다각적 분석 근거를 제공
03

YOLOv8-p2

소형 객체 대응 / 고난도 장면 보조 분석

  • 작은 객체 탐지에 강점을 기대할 수 있는 구조
  • 멀리 있거나 크기가 작은 낙하물 검토에 활용 가능
  • 향후 고난도 탐지 기능 확장용 후보 모델로 의미가 있음
